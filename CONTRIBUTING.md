# Contributing to Telegram GramPro

We welcome contributions to make Telegram GramPro even better! Please read this guide before submitting pull requests or reporting issues.

## ğŸ¤ How to Contribute

### **Reporting Issues**
Before creating an issue, please:
1. Check if the issue already exists in our [GitHub Issues](https://github.com/sadiakant/n8n-nodes-telegram-grampro/issues)
2. Search the [Troubleshooting Guide](TROUBLESHOOTING_GUIDE.md) for solutions
3. Provide detailed information about your environment and the problem

When reporting an issue, include:
- n8n version and node version
- Operating system
- Error messages (exact text)
- Steps to reproduce the issue
- Relevant log files (with sensitive data removed)

### **Submitting Pull Requests**
1. **Fork the repository**
2. **Create a feature branch** from the `main` branch
3. **Make your changes** with proper TypeScript types
4. **Add tests** for new functionality
5. **Update documentation** for any new features
6. **Test thoroughly** before submitting
7. **Submit a pull request** with a clear description

### **Code Standards**
- Use TypeScript for type safety
- Follow existing code patterns and naming conventions
- Add comprehensive error handling
- Include proper documentation and comments
- Write tests for new functionality
- Ensure backward compatibility when possible

### **Development Setup**
```bash
# Clone your fork
git clone https://github.com/your-username/n8n-nodes-telegram-grampro.git

# Install dependencies
npm install

# Start development mode
npm run dev

# Build for production
npm run build
```

### **Testing**
Before submitting changes:
1. Test your changes thoroughly
2. Ensure existing functionality still works
3. Run the build process successfully
4. Test with different n8n versions if possible
